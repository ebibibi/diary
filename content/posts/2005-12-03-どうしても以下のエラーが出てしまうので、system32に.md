```
PHP Warning:  PHP Startup: Unable to load dynamic library 'c:\\php\\ext\\php_mysqli.dll
```

今度はextension=php\_mysqli.dllをphp.iniに書かなければきちんと起動するが、かくと起動しない状態になってしまった。 Google先生に聞いて以下の通りに実行

```
ようやく悟った。PHP 5.0 + MySQL 4.1 + Apache 2.0 で、Apache の起動時にmysql.dll と mysqli.dll がないというエラーが出たとき、次のように確認すればうまくいくと思います。
php.ini の確認
extension_dir にきちんと mysql.dll, mysqli.dll があるかどうか確認。
extension=php_mysqli.dll と extension=php_mysql.dll がコメントアウトされているかどうか確認。
MySQL の確認
そもそも MySQL サーバが起動しているかどうか確認する。
これで駄目な場合、僕は次のようにしています。
%SYSTEM%（多くの場合、C:￥WINDOWS￥SYSTEM32）
システムフォルダに、mysql.dll, mysqli.dll をコピーして、更に libmysql.dll もコピーする。
%WINDIR%（C:￥WINNT か、C:￥WINDOWS）
```
