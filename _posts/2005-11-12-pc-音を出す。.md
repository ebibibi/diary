取り合えず中身に何が入っているのかを把握してなかったので、サウンドデバイスを検索。取り合えず、[Aureal VORTEX8810:AC97準拠（PCI接続、オンボード）](http://www.sony.jp/products/Consumer/PCOM/PCV-L720BP/spec.html)ということらしい。 やりかたはよーわからんので、とりあえずDebian GNU/Linux スレッドテンプレで検索。[これ](http://debian.fam.cx/index.php?Software#content_1_48)の通りにやってみる。

```
vaio:/# echo a > /dev/dsp
bash: /dev/dsp: そのようなデバイスはありません

```

ふむ。

```
apt-get install alsa-base

```

これだけだとalsaconfがはいらなかったのでalsa-utilsもインストール

```
apt-get install alsa-utils

```

ほんでもってalsaconfを実行するも、自動では検出されませんでした。。むぅん？ カーネルが古いのかなぁということでカーネルを新しくしようとしてみる。

```
apt-get -s install linux-image
aptitude install linux-image-2.6.12-1-686

```

途中でLILOのコンフィグをいじれといわれたのでいじってみた。 それで、再起動したら、無事alsaconfでサウンドカードが認識されました。alsaはカーネル2.6が前提なのね。 でも、まだ、Gnomeについてたtotem Movie Playerからは音が出ない。 とりあえず、esdを実行させておけば音は出るもよう。 OS←→alsa←→esd←→アプリケーション という処理のながれなのかな？？
